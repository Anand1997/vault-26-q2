- # Single responsibility principle #srp 
	- There should never be more than one reason for a class to change. Every class should have only one responsibility.
- # Open-closed principle #ocp 
	- The softer entities should be open for extension, but closed for modification.
- # [[Liskov]] substitution principle #lsp 
	- The function that use pointers or references to base classes must be able to use pointers or references of derived classes without knowing it.
- # Interface segregation principle #isp 
	- Clients should not be forced to depend upon interface methods that they do not use.
- # Dependency inversion principle #dip 
	- Depend on abstractions, not concretes.
